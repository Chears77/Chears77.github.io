一、选择github的主页设置

GitHub Pages 搭建个人主页完整手把手教程（你已登录 GitHub，从首页开始操作）
第一步：创建专属网站仓库（最关键，名字不能错）

　　新建仓库填写页面

  1. 页面右上角，点 + 下拉箭头 → New repository（新建仓库）

  2. 仓库名称严格填写：你的 GitHub 用户名.github.io

    • 举例：你 GitHub 账号叫 lisi123，仓库名必须是 lisi123.github.io，一字不差、大小写匹配

  3. 配置选项：

    • Visibility（可见性）：选 Public（公开）（私有仓库免费版不能开 Pages）

    • Add a README file：可以勾选（方便备注）

    • Add .gitignore/ License：全部选 None 不用管

  4. 点击最下方 Create repository 创建仓库

第二步：上传首页文件 index.html（两种方法，新手推荐网页上传）
方式 1：网页直接创建文件（不用装 Git 软件，零门槛）

　　仓库内添加文件按钮

  1. 进入刚建好的 xxx.github.io 仓库主页

  2. 点击绿色按钮 Add file → Create new file

  3. 文件名输入：index.html（必须叫这个，网站默认打开它）

  4. 下方大编辑框粘贴下面基础测试代码：

　　html

　　预览

<!DOCTYPE html>
<html lang="zh-CN">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>我的个人主页</title>
    <style>
        body{text-align:center;margin-top:100px;font-family:微软雅黑;}
    </style>
</head>
<body>
    <h1>欢迎来到我的个人主页</h1>
    <p>这是用GitHub Pages免费搭建的网站</p>
</body>
</html>

  1. 滑到最底部，Commit changes（提交），直接点 Commit new file 保存文件

方式 2：本地写好文件批量上传（适合后续多文件：css、图片、js）

  1. 本地电脑建好文件夹，里面放 index.html、样式 css、图片等所有网页素材

  2. 仓库页面点 Add file → Upload files

  3. 把本地文件夹里全部文件拖进上传框，底部提交保存

第三步：开启 GitHub Pages 部署功能

  1. 在仓库顶部导航栏，点击 Settings（设置）

  2. 左侧菜单栏往下滑，找到 Pages 点开配置页

  3. Build and deployment（部署来源）设置：

    • Source 选择：Deploy from a branch

    • Branch（分支）：选 main

    • Folder（文件夹）：选 /(root) 根目录

  4. 点击蓝色 Save 保存配置

第四步：访问你的个人网站

  1. 保存后等待 1–3 分钟（GitHub 后台打包部署）

  2. 浏览器打开地址：https://你的用户名.github.io

  3. 刷新页面就能看到你写的 index.html 内容；如果 404，多等一会再刷新

五、后续美化 & 更新网站

  1. 修改页面内容

    • 网页端：点开 index.html，点铅笔编辑图标，改完提交

    • 熟练后可以装 Git，本地改完一键推送更新

  2. 套好看模板（不用手写 HTML） 网上搜 GitHub Pages 免费模板（Jekyll 模板、静态 HTML 模板），下载全部文件替换仓库里的文件，重新提交即可一键换皮肤

    1. 区分两个 “主页” 别搞混

        1. xxx.github.io 是独立网站（浏览器打开的完整网页站点）

        1. 和用户名同名仓库的 README，是你 GitHub 账号头像下面的简介卡片（只是资料展示，不是独立网址网站）

常见踩坑提醒

  1. 仓库名字写错、少 .github.io → 网站无法生成

  2. 文件不叫 index.html、放在子文件夹里 → 打开 404 空白

  3. 仓库设为 Private 私有 → Pages 功能直接不可用

  4. 刚保存配置立刻访问大概率 404，一定要等几分钟部署完成

三、本次搭建个人网页全过程总结 & 所有踩坑点避雷清单

（一）搭建流程回顾

  1. 新建专属仓库 用户名.github.io，设为 Public 公开

  2. 下载静态 HTML 模板 MeFamily，本地解压

  3. 因网页单次上传文件超 100 个报错，改用 zip 压缩包上传仓库

  4. 打开 Codespaces 云编辑器，终端执行unzip web.zip解压模板，覆盖原有index.html

  5. 提交解压后的全部文件，仓库根目录出现完整网站资源

  6. 仓库 Settings 开启 GitHub Pages，分支 main、根目录部署，成功访问网站

  7. 尝试本地 Git 同步：克隆仓库→本地修改 index→git 提交推送，遇到网络、登录、命令输入多重问题

（二）全程踩坑点 + 对应避雷方案
1. 网页上传文件数量限制坑

  • 踩坑：一次性拖拽全部模板文件，提示文件超过 100 个无法上传

  • 避雷： 方案 A（网页直传）：分两批上传，先传 assets、forms 文件夹，再传所有 html 文件；方案 B（压缩包）：打包 zip 上传，用 Codespaces 终端命令解压，适合文件量大的模板。

2. 压缩包解压目录错误坑

  • 潜在大坑：打包时把外层MeFamily文件夹一起打包，解压后所有文件嵌套在子文件夹里，访问网站 404

  • 避雷：打包前进入 MeFamily 内部，全选里面所有文件 / 文件夹再压缩，保证解压后index.html直接在仓库根目录。

3. Codespaces 右键无解压按钮坑

  • 踩坑：新版界面移除右键解压选项，找不到解压入口

  • 避雷：直接用终端命令 unzip web.zip，一键解压，出现覆盖提示输入A全部替换。

4. Git 克隆网络中断坑

  • 踩坑：git clone直连 GitHub 报 Connection was reset 连接重置

  • 避雷：克隆地址前加国内加速代理前缀；切换手机热点重试；嫌麻烦直接用网页上传，不用 Git 克隆。

5. 复制命令带乱码特殊符号报错

  • 踩坑：复制教程里带$、^[[200~标记的整行内容，粘贴后系统识别不到 git 命令

  • 避雷：只复制纯命令文字，手动输入或单独复制指令，不要连带行号、提示符一起粘贴。

6. GitHub 令牌找不到入口坑

  • 踩坑：点开仓库的 Settings，翻到底也没有 Developer settings

  • 避雷：必须点右上角个人头像下拉框的 Settings（个人全局设置），不是仓库设置；可直接打开直达链接 https://github.com/settings/tokens。

7. Git 推送登录超时警告

  • 踩坑：git push 出现凭证自动探测超时 warning 弹窗，登录卡顿

  • 避雷：执行配置命令固定登录模式消除警告；长期使用提前生成 classic 令牌，密码栏粘贴令牌而非账号密码。

8. Everything up-to-date 推送无更新坑

  • 踩坑：本地改完文件直接 git push，提示全部已是最新，线上无变化

  • 避雷：修改文件后必须完整执行三步：git add . → git commit -m "修改备注" → git push；缺少 add/commit 不会生成新提交，自然无法上传。

9. Pages 部署基础规则坑

  • 隐性风险：仓库私有、仓库名写错、index.html 不在根目录，打开网址 404 空白

  • 避雷：仓库名严格为用户名.github.io、必须 Public、网站首页文件放在仓库第一层，不嵌套文件夹。

四、后续两种网页更新操作流程（按需选择，以后直接照着做）

方案 A：少量修改，零 Git 命令（小白首选，无任何坑）

　　适合：只改文字、单张图片、偶尔更新

  1. 本地修改好index.html/ 图片 /css 文件

  2. 打开 GitHub 仓库页面 → Add file → Upload files

  3. 只拖拽你修改过的那几个文件，不用全传

  4. 底部填写修改备注，点击 Commit changes 覆盖线上文件

  5. 等待 2~5 分钟，刷新 https://chears77.github.io 查看更新

方案 B：大量修改 / 频繁更新，本地 Git 同步（适合长期维护）
首次前置（仅做一次）

  1. 使用加速链接克隆完整仓库到本地电脑

  2. 配置全局用户名、邮箱，生成 GitHub 令牌用于登录

  3. 执行命令消除凭证超时警告

每次更新标准固定三步（改完文件后执行）

  1. 打开本地仓库文件夹，空白处右键 Git Bash Here

  2. 标记所有改动文件

　　bash

　　运行

git add .

  1. 填写本次修改说明，生成本地提交

　　bash

　　运行

git commit -m "更换模板2"

  1. 同步推送至 GitHub 线上仓库

　　bash

　　运行

git push

  1. 等待几分钟，刷新个人网站查看效果

补充同步指令

　　如果你在 GitHub 网页手动改了内容，本地文件落后，执行这条拉取线上最新文件：

　　bash

　　运行

git pull

　　同步命令简化

　　git add . git commit -m "更新了1" git push
