　　很早以前就想搭建一个个人主页，基于两个方面考虑，一直没有实现。一个是学习成本比较高，以前尝试过用 PHP，但是需要很强的代码能力，包括前端和后端。第二方面是没有一个站点可以免费发布。也就是说当时用 PHP 通过模板生成的网站只能在本地运行，不能够放在网上让别人可观可看。随着 AI 的崛起，我又重新将这个事情纳入了考虑。于是我通过豆包和 GitHub 实现了我之前的想法，我做的网页如下图。我把我的尝试过程和解决方案发出来，方便大家学习。

一、和豆包对话过程（不断迭代和试错）

迭代1：选择托管平台

　　我的问题：帮我找找免费静态网页托管平台。

　　豆包回答：国内可用 Gitee Pages、Zeabur 免费托管静态网页，海外选 GitHub Pages、Cloudflare、Netlify，均无需备案，支持直接上传部署。

迭代2：通过GitHub搭建个人网页完整过程

　　我的问题：我想用我的github做我的一个个人主页，你帮我实现，教我。我已经注册并登录了github了，进入主页后需要做什么。

　　豆包回答：

　　GitHub Pages 搭建个人主页完整手把手教程（你已登录 GitHub，从首页开始操作）

　　一、第一步：创建专属网站仓库（最关键，名字不能错）

　　新建仓库填写页面

  1. 页面右上角，点 + 下拉箭头 → New repository（新建仓库）

  2. 仓库名称严格填写：你的 GitHub 用户名.github.io

    • 举例：你 GitHub 账号叫 lisi123，仓库名必须是 lisi123.github.io，一字不差、大小写匹配

  3. 配置选项：

    • Visibility（可见性）：选 Public（公开）（私有仓库免费版不能开 Pages）

    • Add a README file：可以勾选（方便备注）

    • Add .gitignore/ License：全部选 None 不用管

  4. 点击最下方 Create repository 创建仓库

　　二、第二步：上传首页文件 index.html（两种方法，新手推荐网页上传）

　　方式 1：网页直接创建文件（不用装 Git 软件，零门槛）

　　仓库内添加文件按钮

  1. 进入刚建好的 xxx.github.io 仓库主页

  2. 点击绿色按钮 Add file → Create new file

  3. 文件名输入：index.html（必须叫这个，网站默认打开它）

  4. 下方大编辑框粘贴下面基础测试代码：

　　html

　　预览

<!DOCTYPE html>
<html lang="zh-CN">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>我的个人主页</title>
    <style>
        body{text-align:center;margin-top:100px;font-family:微软雅黑;}
    </style>
</head>
<body>
    <h1>欢迎来到我的个人主页</h1>
    <p>这是用GitHub Pages免费搭建的网站</p>
</body>
</html>

  1. 滑到最底部，Commit changes（提交），直接点 Commit new file 保存文件

　　方式 2：本地写好文件批量上传（适合后续多文件：css、图片、js）

  1. 本地电脑建好文件夹，里面放 index.html、样式 css、图片等所有网页素材

  2. 仓库页面点 Add file → Upload files

  3. 把本地文件夹里全部文件拖进上传框，底部提交保存

　　三、第三步：开启 GitHub Pages 部署功能

  1. 在仓库顶部导航栏，点击 Settings（设置）

  2. 左侧菜单栏往下滑，找到 Pages 点开配置页

  3. Build and deployment（部署来源）设置：

    • Source 选择：Deploy from a branch

    • Branch（分支）：选 main

    • Folder（文件夹）：选 /(root) 根目录

  4. 点击蓝色 Save 保存配置

　　四、第四步：访问你的个人网站

  1. 保存后等待 1–3 分钟（GitHub 后台打包部署）

  2. 浏览器打开地址：https://你的用户名.github.io

  3. 刷新页面就能看到你写的 index.html 内容；如果 404，多等一会再刷新

　　五、后续美化 & 更新网站

  1. 修改页面内容

    • 网页端：点开 index.html，点铅笔编辑图标，改完提交

    • 熟练后可以装 Git，本地改完一键推送更新

  2. 套好看模板（不用手写 HTML） 网上搜 GitHub Pages 免费模板（Jekyll 模板、静态 HTML 模板），下载全部文件替换仓库里的文件，重新提交即可一键换皮肤

    1. 区分两个 “主页” 别搞混

        1. xxx.github.io 是独立网站（浏览器打开的完整网页站点）

        1. 和用户名同名仓库的 README，是你 GitHub 账号头像下面的简介卡片（只是资料展示，不是独立网址网站）

　　常见踩坑提醒

  1. 仓库名字写错、少 .github.io → 网站无法生成

  2. 文件不叫 index.html、放在子文件夹里 → 打开 404 空白

  3. 仓库设为 Private 私有 → Pages 功能直接不可用

  4. 刚保存配置立刻访问大概率 404，一定要等几分钟部署完成

迭代5：使用git工具将模板部署到我的github上

　　我的问题：有没有方法本地修改我的主页程序后自动上传呢，让线上和本地同步修改，有没有这个功能。

　　豆包回答：有的，用 Git 工具就能实现本地改完一键同步线上，本地和 GitHub 仓库双向同步，改完敲两行命令就自动上传更新，下面给小白一套全程傻瓜安装配置步骤。

　　一、第一步：安装 Git 软件（Windows 系统）

  1. 打开 Git 官网下载：https://git-scm.com/download/win

  2. 运行安装包，全程全部点下一步默认安装，不用修改任何设置

  3. 安装完成后，在电脑空白处右键，能看到两个新选项：Git Bash Here、Git GUI Here，代表安装成功

二、第二步：配置你的 GitHub 账号信息（只做一次）

  1. 随便找一个电脑空白文件夹，右键 → Git Bash Here，弹出黑色命令窗口

  2. 依次复制粘贴下面两行，分别替换成你自己的信息，每行输完按回车

　　bash

　　运行

git config --global user.name "Chears77"

　　bash

　　运行

git config --global user.email "你注册GitHub用的邮箱"

　　这一步是绑定你的提交身份，只需要设置一次。

三、第三步：把线上仓库完整克隆下载到本地

  1. 回到你的 Chears77.github.io 仓库主页，点绿色 Code 按钮

  2. 复制 HTTPS 地址：https://github.com/Chears77/Chears77.github.io.git

  3. 在电脑里新建一个专门放网站的文件夹（比如命名 本地主页文件），进入文件夹，右键打开 Git Bash Here

  4. 粘贴克隆命令，回车下载全部文件：

　　bash

　　运行

git clone https://github.com/Chears77/Chears77.github.io.git

  1. 下载完成，文件夹里会多出一个 Chears77.github.io 文件夹，里面就是完整网站源码（和线上仓库一模一样）

四、日常：本地修改 → 一键同步线上（核心操作）

　　1. 本地修改内容

　　打开 Chears77.github.io 文件夹，用记事本 / VS Code 打开 html、图片随便修改，所有改动只在本地电脑，不会影响线上网站。

　　2. 改完三步命令推送线上同步

  1. 在 Chears77.github.io 文件夹内部空白处，右键打开 Git Bash Here

  2. 第一步：标记所有改动文件

　　bash

　　运行

git add .

  1. 第二步：写本次修改备注（随便写，比如 “修改首页自我介绍”）

　　bash

　　运行

git commit -m "修改首页自我介绍文字"

  1. 第三步：推送到 GitHub 线上仓库，网站自动更新

　　bash

　　运行

git push

　　推送成功后，等待 1-3 分钟，你的 https://chears77.github.io 网站就刷新成修改后的内容。

五、反向同步：线上改了内容，拉回本地

　　如果有时候你在 GitHub 网页在线改了文件，本地文件落后了，打开 Git Bash 输入拉取命令一键同步：

　　bash

　　运行

git pull

　　执行完本地文件就和线上完全一致。

六、极简通俗总结

  1. git clone：第一次把线上整个网站下载到电脑（只运行 1 次）

  2. 平时改内容：本地改文件 → git add . → git commit -m "备注" → git push 三步上传

  3. 线上更新过：git pull 拉最新文件到本地

    1. 首次git push可能弹窗让你登录 GitHub 账号，输入账号 + 个人访问令牌（替代密码），令牌获取教程我可以发给你。

    2. 不用担心改错：所有提交都有记录，随时可以回退到任意历史版本。

零基础搭建个人主页【完整实操步骤】

　　全程无门槛，跟着步骤一步步操作，人人都能成功！

步骤1：准备工具与账号

  1. 注册GitHub账号（免费）；

  2. 电脑安装Git工具（用于文件推送同步）；

  3. 下载一套纯静态HTML响应式主页模板。

步骤2：创建专属主页仓库（核心关键）

  1. GitHub右上角点击「New repository」新建仓库；

  2. 仓库名称必须严格填写：用户名.github.io（例如：chears77.github.io）；

  3. 选择 Public 公开仓库，勾选初始化文件；

  4. 创建完成后，复制仓库HTTPS地址。

步骤3：本地搭建网站文件夹

  1. 电脑新建专属文件夹，作为本地网站根目录；

  2. Git克隆远程仓库到本地：git clone 仓库地址；

  3. 将下载好的模板内层所有文件，全部复制粘贴到本地根目录（保证index.html在最外层）。

步骤4：自定义修改主页内容

  1. 用记事本/VS Code打开 index.html；

  2. 自由修改主页文字、介绍、个人信息、页脚、配色；

  3. 替换images文件夹内的图片，自定义专属主页素材；

  4. 可自由新增页面按钮、功能模块，布局完全自定义。

步骤5：Git推送文件到线上仓库

  1. 文件夹内打开Git Bash；

  2. 依次输入三步指令完成推送：

　　git add .

　　git commit -m "更新个人主页内容"

　　git push

步骤6：开启GitHub Pages部署网站

  1. 打开GitHub对应仓库，进入 Settings；

  2. 下滑找到 Pages 部署选项；

  3. 部署来源选择：Deploy from a branch；

  4. 分支选择 main，目录选择根目录 /(root)；

  5. 保存设置，等待2-5分钟自动部署完成。

最终成型：网站仓库树状结构（标准规范版）

新手总结避坑指南（必看）

  1. 所有文件禁止中文命名，是图片空白、样式错乱的最大原因；

  2. index.html 必须在根目录，绝对不能嵌套文件夹；

  3. 大批量更新模板优先使用Git缓存配置，避免推送超时；

  4. 页面更新不生效，一定用 Ctrl+F5强制清缓存；

  5. 追求稳定优先用Gitee Pages，国内访问秒开，无推送报错。
