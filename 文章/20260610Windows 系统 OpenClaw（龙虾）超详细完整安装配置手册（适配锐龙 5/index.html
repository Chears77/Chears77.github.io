Windows 系统 OpenClaw（龙虾）超详细完整安装配置手册（适配锐龙 5 4600U 16G 轻薄本｜2026 最新稳定版）

前言适配说明

　　本文完全结合你本人硬件（AMD R5 4600U、16G 内存、核显 2G、轻薄办公本）踩坑经验撰写，规避所有配置报错、路径错误、模型兼容、开机自启、工具调用失效问题；区分<strong>云端 API 稳定办公方案（你的硬件最优）与本地 Ollama 轻量化方案（硬件受限仅轻量试用）</strong>，明确告诉你硬件适配边界，杜绝网上通用教程带来的卡顿、会话超时、无法操控本地文件问题。OpenClaw 核心定位：本地 AI 智能体，主打电脑办公自动化、终端脚本执行、文件批量处理、浏览器联动；轻薄本优先「豆包生成脚本 + OpenClaw 终端执行」组合，放弃本地大模型全自动工具调用，兼顾流畅度与自动化能力。

一、硬件 & 系统前置要求（必看，决定后续使用体验）

1. 最低硬件门槛（你的设备达标）

　　表格

项目 要求 你的设备状态
CPU 双核以上，支持 VT-x 虚拟化 R5 4600U 6 核 12 线程，达标
内存 最低 8GB，推荐 16GB 16GB 可用 13.9GB，完美适配轻量负载
硬盘空闲 系统盘≥10GB SSD 477GB 固态，剩余充足空间
显卡 无强制独显，本地模型仅核显会减速 AMD 核显 2G，仅适合 3B 及以下极小模型

2. 系统版本要求

　　Windows10 20H2 及以上 / Windows11 全版本，PowerShell 5.0 及以上；必须开启硬件虚拟化（BIOS 内 AMD-V），否则后续 Ollama 本地模型无法运行。

3. 预装三大核心依赖（缺一不可，90% 新手报错源于依赖缺失）

依赖 1：Node.js v22 LTS（官方强制最低版本，低于 22 直接安装失败）

  1. 官网下载：https://nodejs.org/ 选择 22.x 长期支持版 64 位 msi 安装包

  2. 安装关键设置：全程默认，务必勾选 Add to PATH（自动写入环境变量，禁止取消），安装路径默认 C 盘即可，无需修改

  3. 安装完成验证：管理员 PowerShell 分别输入两条命令，输出版本号即成功

　　powershell

node -v
npm -v

　　要求输出v22.x.x，版本低于 22 重新安装。

依赖 2：Git（拉取插件、版本更新必备）

  1. 官网下载：https://git-scm.com/download/win

  2. 全程默认安装，PATH 选择「Git from the command line and also from 3rd-party software」

  3. 验证：git --version 输出版本号正常

依赖 3：Python3.10+（后续脚本运行、视频下载、表格处理必备）

  1. 官网下载 Python3.11 稳定版，安装首界面勾选 Add Python to PATH

  2. 后续 OpenClaw 终端运行 Excel 处理、yt-dlpB 站下载脚本全依赖此环境

二、OpenClaw 一键正式安装（管理员 PowerShell 执行，官方国内稳定脚本）

步骤 1：权限前置修复（Windows 默认策略阻止远程脚本执行，必执行）

  1. 右键开始菜单，选择Windows PowerShell (管理员)

  2. 执行执行策略放行命令，输入 Y 确认

　　powershell

Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser

步骤 2：国内镜像一键安装（避免 GitHub 超时下载失败）

　　powershell

iwr -useb https://open-claw.org.cn/install-cn.ps1 | iex

  • 等待脚本自动下载、全局 npm 安装 OpenClaw，全程不要关闭窗口

  • 出现🦞龙虾欢迎标语、版本号即安装完成（当前稳定版 2026.6.8）

步骤 3：基础安装验证

　　新开普通 PowerShell，输入版本查询：

　　powershell

openclaw --version

　　正常输出版本号，代表本体安装完成。

四、适配轻薄本的最优使用范式（解决聊天卡顿、无法创建文件夹痛点）

核心分工方案（彻底规避云端 API 本地权限缺陷、硬件性能不足）

  2. 回到 OpenClaw 网页内置终端，统一运行脚本，拥有完整本地读写权限，可操作桌面、D 盘、系统文件

  3. 适用场景：批量创建文件夹、Excel 数据拆分、Word 周报生成、B 站视频下载、文件自动分类归档

八、高频报错排查 & 避坑总结（你踩过的坑集中解决方案）

报错 1：tools.profile 填写 agent 配置崩溃

　　合法固定值仅四个：minimal/coding/messaging/full，只能选 coding（完整文件权限），禁止自定义值。

报错 2：云端模型提示does not support tools无法调用文件

　　本质：云端 API 架构隔离，远程模型无本地权限，放弃 AI 直接调用文件，改用豆包生成脚本 + 终端执行，唯一稳定解法。

报错 3：会话 stalled 卡死、推理超时

　　原因：本地模型 CPU 算力不足、云端长上下文超时；解决：放弃本地大模型，全程云端 Lite 闲聊，脚本独立终端运行。

报错 4：JSON 配置解析失败

　　必须 UTF-8 编码保存、provider 名称禁止带/层级符号（禁止 ollama/local，只能写 ollama），路径反斜杠转义\\。

报错 5：模型自动简写「桌面」路径，创建文件夹失败

　　不要依赖 AI 直接创建文件，统一脚本执行，彻底规避路径简写校验拦截问题。

九、完整落地工作流示例（猪肉行业周报自动化示范）

  1. 豆包生成：Edge 搜索下载 Excel 脚本、Excel 合并 + Word 汇总 Python 脚本

  2. 启动 OpenClaw 网关，打开内置终端，切换桌面目录

  3. 终端运行下载脚本，手动筛选下载行业报表 Excel

  4. 终端运行 Python 汇总脚本，自动建文件夹、合并表格、生成 D 盘 Word 周报

  5. 全程仅用龙虾终端执行，不用切换系统 PowerShell，统一日志、统一文件权限

十、最终使用取舍建议（贴合你的轻薄本硬件）

  1. 纯聊天：优先豆包网页 / OpenClaw 免费 V2-Lite，流畅零成本；

  2. 电脑自动化：豆包写代码，OpenClaw 终端执行，发挥本地权限核心优势，完美规避云端 API 短板；
