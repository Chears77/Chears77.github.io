> 生成器：`law-knowledge-base/scripts/gen_website.py`（改动后需重建 `gen_website.py` 才持久化）

## 一、已落地的 4 项功能

| # | 需求 | 实现方式 | 关键函数 |
|---|------|----------|----------|
| 1 | 左侧层级点击筛选 | 点「法律/司法解释/…」→ 中间只显该层级并高亮（蓝底加粗）；点根节点「工建法研·工程建设法规库」→ 清除筛选回总库全量 | `applyBrowseLevel` / `clearBrowseLevel` |
| 2 | 视图 + 排序控件 | 视图：列表(默认)/卡片；排序：时间逆序(默认)/时间正序/相关性/名称/条文数 | `switchView` / `sortLaws` / `relScore` |
| 3 | 下载按钮 | 「法规库」折叠按钮左侧的 ⬇ 按钮；下载**当前筛选/检索范围**；支持 每部独立.md(ZIP) / 合并.md / Word / PDF(打印) | `downloadLibrary` / `makeZip` |
| 4 | 库筛选按钮 | 默认「总库」；下拉含 住建/市政给排水/交通/招投标/建设单位常用/施工单位常用/地方 + 自选库（勾选层级/地区/关键词）；菜单显示每库数量 | `selectLibrary` / `openCustomLib` / `libMatch` / `libCount` |

> 设计要点：层级筛选与库筛选是**叠加**生效的（先层级后库，再排序）；下载、库菜单都与"当前可见范围"联动，所见即所下。

## 二、库分类的推敲（重点：水利库问题）

用户最初提议「住建库、水利库、招投标库、建设单位常用库、施工单位常用库、地方库」。
经对 373 部法规真实数据逐条核验，发现一个**关键事实**：

- **库内目前没有任何"水利（水库/堤防/河道/灌区）"类法规。**
- 标题/领域里含"水"的法规共 13 部，全部是 **给排水 / 防水 / 燃气 / 园林绿化**（如《城镇排水与污水处理条例》《建筑与市政工程防水通用规范》《城市给水工程项目规范》），属市政/住建范畴。
- 用「水利部」作发文单位匹配到的 9 部，其实是**招投标法规**（八部委联合发文里列了水利部），并非水利专业法规——属误报，已排除在匹配逻辑之外（只匹配标题+领域，不匹配发文单位）。

**结论与处理：** 把空壳的「水利库」替换为真实存在的「**市政给排水库**」（匹配 排水/给水/供水/防水/水务/污水/海绵/燃气/园林绿化），命中 13 部，内容名副其实。若日后补充水利工程法规，自选库可直接用关键词「水利」检索，无需改代码。

## 四、三项关键设计决策与理由

1. **分类用"规则自动归类"而非改数据**
   直接对 373 部数据的 `level/field/region/title` 做正则判定，零侵入、零维护成本；新增法规自动归入对应库。代价：正则需随领域关键词演进微调（如本次水利→市政给排水的修正）。

2. **相关性排序用"启发式评分"而非接入大模型**
   `level` 权重（法律/司法解释 > 行政法规 > 规章 > 其他）+ `field/title` 命中核心词库（施工/招标/质量/安全/合同…）加权。离线、秒级、零成本；代价：不是语义级理解，但覆盖了"工程建设相关性"的绝大部分诉求。

3. **下载用"纯 JS 写 ZIP" + md/word/pdf 三格式**
   不依赖任何外部库/网络，双击 HTML 即可用；`makeZip` 用 `TextEncoder + Uint8Array` 拼字节（含中文文件名 UTF-8 标志位），已用 Python `zipfile` 验证可正常解压。PDF 走浏览器打印（用户可自行选另存为 PDF）。

## 五、给后续迭代的建议

- **补数据优先**：交通库(7)、市政给排水库(13)体量少，是因为对应领域法规尚未录入，而非功能问题——继续用 `law-kb-expand` 技能拓库即可自然充实。
- **水利库**：待录入《水法》《防洪法》《水利工程质量管理办法》等后，可恢复"水利库"或在自选库关键词搜"水利"。
- **相关性词库**：若发现某些重要法排不进前列，在 `CORE_KW` 增补关键词即可，无需重构。
- **下载性能**：当前为浏览器端同步生成，373 部一次性打包在普通机器上约 1–2 秒，可接受；若未来上千部，可改为分块/Web Worker。
