# 法规录入指南（日常工作流程）

目标：**你只负责提供内容并起好文件名，工具负责把文件放进正确文件夹。**

## 一、日常流程（两步搞定）

1. **让 AI 生成规范 md，放进 `0新录入`**
   - 把本《录入指南》+《格式说明》一起交给任意 AI，附上法规的链接 / PDF / 图片；
   - 让 AI 按规范生成 md，并保存到 `2法规库\0新录入\` 文件夹。
   - （也可以手动写，或手动把 md 直接放进 1~12 对应文件夹。）
2. **双击 `rebuild.bat` 一键上线**
   - 脚本自动完成：① 把 `0新录入` 的文件按 `level` 归位到 1~12 文件夹 → ② 校验 level 匹配 / 重复 → ③ 重建检索索引、阅读站、导航门户 → ④ 同步 `法规清单.xlsx`。
   - 浏览器刷新即可看到新法规。

> 归位只搬文件夹、不改名、不为地方类建子目录；放错文件夹也不怕，rebuild 会自动按 `level` 纠正。
> 想先看会动哪些文件，先跑 `python new_law.py scan`（只读预览）。

## 二、单个文件归位 / 交互录入

- 归位单个文件：`python new_law.py place 某文件.md --go`
- 交互式录入（完全手填，直接生成规范 md 到正确文件夹）：`python new_law.py new`

> ⚠️ 本工具**不做改名**：文件名由你/AI 在生成时定好（约定 `日期-文号-标题.md`）。
> 原因：现有 294 部法规的文件名日期写在文件名里、地方类平铺在 `NN-层级` 文件夹下；
> 自动改名/建子目录会破坏既有结构。若误操作改乱了文件，可运行 `python recover.py --go`
> 依据 `法规清单.xlsx` 记录的原始路径整体恢复。

## 三、标准生成 prompt（给 AI 用，保证规范）

> 复制以下模板，把《XXX》换成具体法规名发给任意 AI 即可。

```
请为《XXX》生成一份符合「工程建设法规AI知识库」规范的 Markdown 文档。

要求：
1. 文档以 YAML Front Matter 开头（用 --- 包裹），必须包含以下字段并填全：
   - title：法规全称
   - level：效力层级，只能取以下之一：
       法律 / 司法解释 / 中央行政法规 / 中央部门规章 / 中央规范性文件 /
       地方行政法规 / 地方规章 / 地方规范性文件 / 标准规范 /
       司法案例 / 行政案例 / 政策解读
   - doc_type：法律/法规/规章/标准/司法解释 填 law；判例/解读 填 doc
   - publisher：发布机构
   - doc_number：发文字号（如「主席令第86号」「法释〔2020〕25号」）；无则留空字符串 ""
   - publish_date / effective_date / revise_date：日期，格式 YYYY-MM-DD（无则留空）
   - field：专业领域（如 招投标 / 施工 / 质量 / 安全），可留空
   - region：仅「地方类」(06/07/08) 必填，写 省 或 省/市（如 四川省 / 四川省/成都市）；其余层级留空
   - status：现行 / 已废止 / 待核实
   - verify_status：固定写 初稿待校核
   - source_url：官方来源链接，可留空
   - tags：标签，可留空
2. 正文结构：
   - 法律/法规/规章/标准/司法解释：用「## 第X章 章名」+「### 第X条 条文」
   - 司法案例/行政案例/政策解读：用「## 分章」+「### 小节」
3. 文件名请命名为 `日期-发文字号-标题.md`（日期取 修订>施行>颁布 中最早有值者，YYYYMMDD；无文号则省略中段）；所在文件夹可任意，入库脚本会按 level 自动归位。
4. 发文字号中的「/」按规范改写为「-」（如 GB/T50319-2013）。
```

## 四、命名与归类规则（工具自动处理，了解即可）

- **文件名**：`日期-发文字号-标题.md`，日期优先级 修订 > 施行 > 颁布（YYYYMMDD）；无文号则省略中段。
- **文件夹**：按 `level` 自动落入 `NN-层级名`；地方类再按 `region` 建 `省` 或 `省/市` 子目录。
- **归类唯一依据**：Front Matter 的 `level` 字段，不是文件夹位置。所以「放错文件夹」不要紧，`new_law.py place` 会按 level 纠正。

## 五、注意事项

- 网页/清单都是「由 md 生成」的派生文件，字段以 md 为准。
- 法规清单.xlsx 只有「备注」列可自由批注（每次重建会保留），其余列会被 md 覆盖。
- 同一部法不要跨文件夹重复存放；`check.py` 会在 rebuild 前拦下重复与 level 不匹配问题。
