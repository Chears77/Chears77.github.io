# 法规文件格式规范（v2.0 · 2026-07-19 调整）

本文件说明工程建设法规知识库中所有 Markdown 文档的统一格式约定。**v2.0 随「2初稿 → 2法规库」12 类效力层级重构同步更新。**

---

## 一、目录组织（12 类效力层级）

源文件统一放在 `2法规库/` 下，按效力层级分为 12 个一级文件夹，命名带序号前缀：

```
2法规库/
├── 01-法律/              （全国人大及其常委会制定）
├── 02-司法解释/          （最高法司法解释、全国人大常委会立法解释）
├── 03-中央行政法规/      （国务院条例、规定）
├── 04-中央部门规章/      （国务院部委令；可按机构细分）
├── 05-中央规范性文件/    （部委通知、意见、办法等，全国适用）
├── 06-地方行政法规/      （省级人大/政府地方性法规、条例；按省份细分）
├── 07-地方规章/          （省级政府规章）
├── 08-地方规范性文件/    （省市规范性文件；按省/市细分）
├── 09-标准规范/          （GB / JGJ 等强制性及推荐性技术标准）
├── 10-司法案例/          （最高法指导性案例、参考性案例）
├── 11-行政案例/          （住建部等行政部门通报的违法违规典型案例）
└── 12-政策解读/          （部委对重要规章、标准的官方政策解读）
```

- 地方类文件夹（06 / 07 / 08）下**当前平铺存放**（与现有 294 部结构一致，**不建省 / 市子目录**）；地区信息由 Front Matter 的 `region` 字段驱动网页「地区树」，文件夹内无需再按地区分子目录。
- 中央类文件夹（03 / 04 / 05）下可按发布机构细分（如 `04-中央部门规章/住建部/`），不强制。

> 文件的 `level` 字段是分类的**唯一权威依据**——生成网站、索引、导航时都按 Front Matter 里的 `level` 归类，而不是按所在文件夹名。文件放在哪个文件夹仅作人工管理用，二者应保持一致。

## 二、文件命名

```
<日期>-<发文字号>-<文件名>.md
```

### 2.1 日期（取其一，优先级从高到低，格式 YYYYMMDD）
1. **修订日期** `revise_date`：法规被修订后重新公布/施行的日期（仅修订版填写，命名优先级最高）。
2. **施行日期** `effective_date`：法规生效日期。
3. **颁布日期** `publish_date`：法规首次公布/发布日期。

> 例：`20210101-主席令第45号-中华人民共和国民法典（合同编·建设工程合同）.md`（施行日期 2021-01-01）；`20200501-国务院令第724号-保障农民工工资支付条例.md`（施行日期 2020-05-01）；`20190302-国务院令第709号-中华人民共和国招标投标法实施条例（2019修订）.md`（修订版施行日期 2019-03-02）。

### 2.2 发文字号
- **有真实发文字号则写** `doc_number`；**找不到真实发文字号的，整段省略**，写成 `<日期>-<文件名>.md`（见 2.4）。
- 发文字号字段本身须遵循 2.3 的"统一 / 缩写"规范。

### 2.3 发文字号统一与缩写规范（重要）
| 情形 | 处理 | 示例 |
|------|------|------|
| 前缀冗余 | `中华人民共和国主席令`→`主席令`；`中华人民共和国国务院令`→`国务院令` | `中华人民共和国主席令第68号`→`主席令第68号` |
| 中文数字 | `主席令〔2017〕第八十六号`→`主席令第86号`（统一阿拉伯数字） | 含"第X号"的中文数字一律转写 |
| 多部门联合发布 | 取首部门＋`等N部门第Y号`（N=部门数，去"令"） | `国家发展和改革委员会、建设部、铁道部、交通部、信息产业部、水利部、中国民用航空总局、国家广播电影电视总局令第2号`→`国家发展和改革委员会等8部门第2号` |
| 长机构名 | `人民代表大会常务委员会`→`人大常委会`；`国家发展和改革委员会`→`国家发改委` | `北京市人民代表大会常务委员会公告〔15届〕第61号`→`北京市人大常委会公告第61号` |
| 冗余注释 | 去掉 `（解读）`、`（指导案例X号）`、`（2015年、2021年修正）`、`（2018修订）`、`·工程建设领域专章` 等说明 | `国务院令第724号·工程建设领域专章（解读）`→`国务院令第724号` |
| 年份＋令 | `X2018年第16号令`→`X令第16号` | `国家发展改革委2018年第16号令`→`国家发展改革委令第16号` |
| 届次 | 去掉 `第十三届` 等（文号已含公告序号） | `河北省第十三届人大常委会公告第6号`→`河北省人大常委会公告第6号` |
| 发改类长号 | 适当删冗余字（`法规发`→`规发` 等） | `晋发改法规发〔2021〕373号`→`晋发改规发〔2021〕373号` |
| **总原则** | **缩写后发文字号原则上 ≤ 15 字**；多部门联合令按上表"等N部门"形式（用户示例 `国家发展和改革委员会等8部门第2号` 共 16 字，作为特例认可） | |

> 说明：`建设部令` 与 `住房和城乡建设部令` 为同一部委不同时期名称，**予以保留区分**（不强行统一）；其余同一发文机关的异写（如 `中华人民共和国主席令` / `主席令`）必须统一。

### 2.4 无发文字号的处理
下列情况视为"找不到真实发文字号"，**文件名省略中段、`doc_number` 置空（`doc_number: ""`）**：
- 占位写法：`未标注（……）`、`载《……》`、原文未标注文号；
- 仅以会议通过：`XX省第X届人民代表大会常务委员会第X次会议通过`；
- 地方性法规无统一文号。

> 例：`20200501-保障农民工工资支付条例.md`（无发文字号，省略中段，仅留日期与文件名）。

### 2.5 文件名非法字符
- 不得包含 Windows 非法字符 `< > : " / \ | ? *`；
- 发文字号/标题中的 `/`（如 `GB/T 50319-2013`）替换为 `-` → `GB/T50319-2013`。

示例（规范后）：
- `20220605-主席令第104号-中华人民共和国噪声污染防治法（2021）.md`
- `20180701-国家发展改革委令第16号-必须招标的工程项目规定.md`
- `20030801-国家发展和改革委员会等8部门第2号-工程建设项目勘察设计招标投标办法.md`
- `20010301-辽宁省人大常委会公告第40号-辽宁省建设工程招标投标条例.md`
- `20200501-保障农民工工资支付条例.md`（无发文字号，省略中段）

## 三、文件头（YAML Front Matter）

每个文件开头以 `---` 包裹的元数据块，字段如下：

| 字段 | 含义 | 示例 |
|------|------|------|
| `title` | 法规全称 | 必须招标的工程项目规定 |
| `level` | 效力层级（见上表 12 类之一） | 中央部门规章 |
| `region` | 地区（中央类留空或 `国家级`；地方类写 `四川`、`四川/成都` 等） | 四川 |
| `publisher` | 发布机构 | 国家发展和改革委员会 |
| `doc_number` | 文号 | 国家发展改革委令第16号 |
| `publish_date` | 发布日期（颁布日期） | 2018-03-27 |
| `effective_date` | 实施日期（施行日期） | 2018-06-01 |
| `revise_date` | 修订日期（修订版填写，命名日期优先级最高） | 2019-03-02 |
| `field` | 专业领域 | 招投标 / 住建 / 造价 / 安全 / 通用 |
| `status` | 状态 | **现行 / 已废止 / 待核实** |
| `superseded_by` | 替代关系 | 《必须招标的工程项目规定》（国家发展改革委令第16号，2018年） |
| `doc_type` | 文档类型 | law（按条切分）/ doc（按章节切分），判例解读类用 doc |
| `source_url` | 官方来源 | https://www.ndrc.gov.cn/ |
| `tags` | 关键词标签 | [必须招标, 规模标准] |
| `verify_status` | 校核状态 | **初稿待校核 / 已校核** |

> 这些字段会被索引构建器读取，用于按层级 / 领域 / 地区 / 状态筛选，也是 AI 问答引用时的出处标注来源。

## 四、初稿校核机制（verify_status · 重要）

- 初稿阶段，所有法条均为 **`初稿待校核`**：条文要义依据官方文本整理，但正式发布 / 对外引用前必须逐条核对政府公报或官网原文（条款号、措辞、金额、日期都可能存在偏差）。
- 索引构建器已将 `verify_status` 写入 `search_index.json`，检索软件对每条显示「初稿待校核」黄色角标，并在页面顶部统一提示，保持诚实、可控。
- 条文经官方原文核对无误后，将对应文件的 `verify_status` 改为 `已校核`，重新构建索引即可更新角标。

> 这是产品的底线：**宁可标注"待校核"，也不假装"绝对准确"**。可信、可追溯、可复核，正是相对"直接问通用 AI"的核心差异。

## 五、正文结构

```markdown
# 法规名称

> 发布机构：XXX ｜ 文号：XXX ｜ 发布日期：XXX ｜ 实施日期：XXX

## 第一章 总则

### 第一条
内容……

### 第二条
内容……

## 第二章 招标

### 第八条
内容……
```

- **切分粒度：按"条"**（`### 第X条` 为最小检索单元，RAG 引用最精准）。法规 / 规章类设 `doc_type: law`，按"条 / 章"切分。
- **"章"层级保留**：用 `## 第X章 标题` 表示，索引构建器会追踪每条所属的章，用于目录导航与引用上下文（如「第一章 总则 · 第五条」）。
- 判例 / 解读类（`doc_type: doc`）按 `## 分章`、`### 小节` 切分。
- 长条文内的分项用 `（一）（二）` 或 `-` 列表。

## 六、已废止法规处理（需单独标记并注明替代关系）

- `status` 设为 `已废止`。
- 必须填写 `superseded_by`，注明被哪一部法规在何时替代。
- 检索结果中，已废止条文以**红色醒目标记**，并展示替代关系，避免误用旧条款。

示例（《工程建设项目招标范围和规模标准规定》）：

```markdown
---
title: 工程建设项目招标范围和规模标准规定
status: 已废止
superseded_by: 《必须招标的工程项目规定》（国家发展改革委令第16号，2018年）
---
```

> 价值说明：旧规（2000年）施工门槛为 200 万元，新规（2018年）上调为 400 万元。
> 通用 AI 可能直接引用旧的 200 万而不提示已废止；本库会明确标注「已废止 → 替代：2018年规定」，
> 这正是"可信、可追溯"相对"直接问豆包"的核心差异。

## 七、RAG 引用示例

用户问："施工单项合同多少金额必须招标？"

系统检索到 `必须招标的工程项目规定.md` 的"第五条"（现行），以及 `工程建设项目招标范围和规模标准规定.md` 的"第五条"（已废止），返回：

> ✅ 依据《必须招标的工程项目规定》（国家发展改革委令第16号）第五条：
> 施工单项合同估算价在 **400 万元人民币以上** 必须招标。
>
> ⚠️ 另检出《工程建设项目招标范围和规模标准规定》第五条（**已废止**），
> 旧标准为 200 万元，已由上述 2018 年规定替代，请以 400 万元为准。
